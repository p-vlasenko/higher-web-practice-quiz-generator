Проблема на скрине вызвана ошибкой в модуле rollup, связанной с несовместимостью
архитектуры процессора (arm64) и npm. Видимо, вы работаете на MacBook с процессором Apple Silicon (arm64), что может вызывать проблемы с некоторыми пакетами, не имеющими нативных сборок для arm64.
Из сообщения об ошибке видно, что не удается найти модуль @rollup/rollup-darwin-arm64.
Это известная проблема npm с опциональными зависимостями (optional dependencies).
[issue](https://github.com/vitejs/vite/issues/16341?utm_source=chatgpt.com)

Решение, предложенное в сообщении об ошибке, следующее:

Удалить package-lock.json и папку node_modules.
Повторно запустить npm install.

```bash
rm -rf node_modules package-lock.json
npm install
```
Если не помогает, можно попробовать более полный сброс кэша и переустановку.
```bash
# Удаляем существующие зависимости
rm -rf node_modules package-lock.json

# Очищаем кэш npm
npm cache clean --force

# Переустанавливаем зависимости
npm i
```
Другой вариант — использовать флаг --ignore-optional при установке, но это может привести к отсутствию необходимых опциональных зависимостей.

Также можно попробовать установить rollup отдельно: npm install rollup@latest, или понизить версию vite.

У меня нет возможности работать с маком, поэтому не могу проверить решение на практике (под линуксом все работает без проблем), если ничего из вышеописанного не помогает, стоит использовать докер с linux, не обязательно собирать образ, достаточно поднять контейнер с linux и nodejs и смонтировать туда проект.

Описанная проблема не решается исключением node_modules из .gitignore, кроме того, это вызовет добавление большого количества файлов в репозиторий, что не приемлемо.

Отмечу что поддержка MacBook не заявлена в требованиях.
